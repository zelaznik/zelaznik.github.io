Cosmo Connect Live × GitHub

Cosmo-Connect is the dating site about nothing. Inspired by OkCupid, with a throwback to Seinfeld. The website is built using Ruby on Rails, PostgreSQL, Javascript, and Backbone, with bootstrap and custom styling. The database schema is fully atomic. Answers to multiple choice questions such as gender and religion are stored as numbers rather than text. Materialized views keep a cache of much of the JSON, which shortens the time needed to compile HTML and Javascript. Postgres triggers make sure that all of the corresponding records in companion tables are kept consistent. Users can look for men, women, or both, thanks to many-to-many relationships. As always, passwords are hashed, using BCrypt.

Frozen Dictionary GitHub

Python's support for immutable objects is inconsistent. Immutable objects offer advantages such as being hashable and thread safe. Immutable objects don't have to be copied before being passed to a new thread. Immutable objects can be used as keys to a dictionary or items in a set. Python's built in library offers immutable arrays, called tuples, and immutable sets, called frozen sets. There is no frozen dictionary, or hash lookup. This library fixes that. The implementation is done in Cython, meaning the library is compiled to pure-C, so the lookup time is within 10 ns of a built in dictionary. A recursive add-in can do a deep-freeze of sets, arrays, and mappings, which is perfect for freezing JSON data, not normally possible in Python.

Ruby Chess GitHub

Classic chess game implemented in the Ruby console. The board displays in blue which squares a given piece can move to. A player clicks on the piece they want to move, and then browses to the square they wish to move the piece. The move is entere if and only if it is valid. Normal chess rules apply, no moving into check, for example. Many pieces share common characteristics, which is where mixins come to play. For example, a king and queen can move in the same directions, so they both include DIRECTION_UNLIMITED, but the king can only move one spot, so it includes MAGNITUDE_ONE, while the queen includes MAGNITUDE_UNLIMITED.
